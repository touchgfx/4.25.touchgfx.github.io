TouchGFX AL設定

如果為專案啟用了TouchGFX Generator，X-CUBE使用者介面就會出現三個主要群組(Display (顯示)、Driver (驅動程式)及Video Decoding (影片解碼))。 第四個群組為「Dependencies」(相依關係)，會在發現目前設定存在問題時出現。

  • Dependencies (相依關係)：此群組用於通知開發人員有關設定中的相依關係、警告或具體錯誤等訊息。 如果沒有項目存在，此群組將會隱藏。
  • Display (顯示)：此群組包含顯示相關設定，例如介面、影像緩衝區位元深度、寬度及高度。 這些設定會直接影響TouchGFX專案的畫布大小，以及為資產所產生的程式碼。
  • Driver (驅動程式)：此群組可讓使用者加入，取得有關應用時標源、圖形加速(DMA2D及GPU2D)和RTOS的現成驅動程式。
  • Additional Features (其他功能)：此群組包含設定中可用的其他功能，例如使用非記憶體映射Flash的圖像和字型資料建立應用程式，或是具有向量渲染功能的應用程式。
  • Video Decoding (影片解碼)：此群組可讓使用者啟用硬體或軟體的影片解碼。 如果要使用影片小工具，此為必要選項。 注意！不是所有的MCU都支援影片解碼。

STM32CubeMX之中的TouchGFX Generator UI

本節將介紹TouchGFX Generator用於設定STM32CubeMX抽象層的各項設定的不同選項。

📄️ 顯示

Display(顯示)群組包含顯示相關設定，例如介面、尺寸及緩衝處理策略。

📄️ 驅動程式

驅動程式一節可協助開發人員為TouchGFX AL的各種功能選擇驅動程式。

📄️ 其他功能

如果專案設定中有任何可用的其他功能，就會出現「Additional Features」(其他功能)群組。

📄️ 影片解碼

「影片解碼」區段可讓開發人員透過硬體或軟體影片解碼功能增強TouchGFX HAL。
